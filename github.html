Github

Introduskjon til samarbeid i GitHub

GitHub har vært en sentral del av samarbeidet i eksamensgruppen. Vi bruker det til å dele kode og holde oversikt over endringer. Siden vi er flere som arbeider på samme prosjekt, har vi blitt enige om å pushe commits med beskrivende meldinger. Dette gjør commit-historikken oversiktlig og gir alle i gruppen en tydelig forståelse av hva som har blitt endret.

Fordeler med GitHub

GitHub har gitt oss mange verktøy som har gjort samarbeidet effektivt. Her er noen av fordelene:

  • Versjonshistorikk: Vi kan spore hvem som har gjort hvilke endringer og når.
  • Branches: Dette lar oss utvikle nye funksjoner uten å påvirke hovedkoden.
  • Pull requests: Gjennomgang av kode før den legges til i hovedprosjektet forbedrer kvaliteten.

Ved å bruke disse funksjonene har vi kunnet jobbe mer strukturert og unngått store feil.

Utfordringer som nybegynnere

Som nybegynnere i GitHub har vi møtt noen utfordringer, spesielt med konflikter i koden. Dette skjedde ofte når flere jobbet på samme filer samtidig, som HTML eller CSS. Små endringer fra ulike medlemmer overskrev hverandre, og dette førte til tap av innhold eller feil.

Tips for å unngå konflikter:

  • Kommuniser tydelig: Avklar hvem som jobber på hva tydelig.
  • Bruk branches: Del opp arbeidet for å unngå endringer i samme del av koden.
  • Gjør små commits: Hyppige og små endringer er enklere å håndtere.
Oppsummering

Til tross for noen startproblemer har GitHub vist seg å være et uvurderlig verktøy for godt samarbeid og effektivitet. Det har både styrket prosjektet og forbedret arbeidsflyten, slik at vi kan jobbe raskere uten å miste kvaliteten på arbeidet.
