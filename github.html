GitHub samarbeid

Effektivt samarbeid i GitHub

GitHub har vært en sentral del av samarbeidet i eksamensgruppen vår. Vi bruker det til å dele kode og holde oversikt over endringer, noe som har gjort samarbeidet mye mer effektivt. Alle i gruppen har blitt enige om å pushe commits med beskrivende meldinger. Dette gjør commit-historikken oversiktlig, og gir alle en tydelig forståelse av hva som har blitt endret.

Versjonskontroll og sporing av endringer

En av de største fordelene med GitHub er muligheten til å spore endringer i prosjektet. Vi kan enkelt se hvem som har gjort hvilke endringer, og når. Dette gir oss muligheten til å utvikle nye funksjoner uten å forstyrre hovedprosjektet, ved å jobbe på separate grener (branches). Når endringene er klare, kan vi gjøre en "pull request" for å gjennomgå og integrere dem i hovedprosjektet.

Fordeler med GitHub

  • Versjonskontroll: Enkel sporing av alle endringer i koden, som gir en oversiktlig commit-historikk.
  • Samarbeid i sanntid: Flere utviklere kan jobbe på samme prosjekt uten å forstyrre hverandre ved hjelp av grener og pull requests.
  • Kvalitetskontroll: Code reviews gir muligheten til å gjennomgå kode før den legges til i prosjektet, noe som forbedrer kvaliteten på arbeidet.
  • Gratis og åpen kildekode: GitHub er gratis for offentlig lagring av prosjekter, og koden er åpen kildekode, noe som fremmer samarbeid og deling av kunnskap.

kodegjennomgang og kvalitetskontroll

Gjennomgang av kode før den legges til i hovedprosjektet har bidratt til å forbedre kvaliteten på arbeidet vårt. Det har vært spesielt nyttig for å unngå feil, og sikre at vi holder en høy standard på koden.

Utfordringer og læring

Som nybegynnere i GitHub har vi møtt utfordringer med konflikter i koden, spesielt når flere jobbet på samme filer. Til tross for noen utfordringer har GitHub vært et uvurderlig verktøy som har styrket samarbeidet vårt og forbedret arbeidsflyten.
